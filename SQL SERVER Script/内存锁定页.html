锁定内存页在数据库中的优点和缺点：


SQL Server 使用VirtualAlloc来分配内存，无内存压力时，SQL Server会尽可能地申请内存来缓存数据，当内存出现压力时，会出现缓存数据频繁地换出/换入，从而导致性能下降。锁定内存页可以使数据库强制占有申请到的内存，即使在操作系统出现内存压力时也不释放，这样保证SQL Server对内存持有性，同时也可能导致操作系统因为内存不足而出现问题，因此在设置锁定内存页时，需设置SQL Server最大内存以保证给操作系统留有足够的内存。
